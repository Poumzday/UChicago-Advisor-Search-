# NBER weekly digest

A menu-bar app that fetches the latest NBER working papers each week, scores them
against your research interests (see `profile.md`), and shows the ones scoring
**above 80** as a dropdown in your top-right menu bar.

- **🔴 NBER N** when you have N unread papers; **⚪ NBER** once everything is read.
- Click the icon for a dropdown; expand a paper for its 5-line summary, then
  "Open paper ↗" (opens the link, marks it read) or "Mark as read".
- The icon lives quietly in the menu bar — it's the "tab on top" that drops down.

There's also a **weather tab**: a condition icon + temperature (°F) in the menu
bar; click it for today's high/low, your city, and the next 10 hours in 2-hour
steps (temperature, condition, and the chance of rain when it's above 25%). It
refreshes hourly.

Pieces:
- `menubar_app.py` — always-on NBER menu-bar app; reads `digest.json`, opens a
  1-page summary per paper (overview / methods / findings, clickable author
  affiliations, link).
- `weather_app.py` — the weather tab (hourly forecast list).
- `bus_app.py` — CTA #6 bus tab: next 3 northbound arrivals at Hyde Park & 53rd
  (live CTA Bus Tracker predictions; refresh on click). Needs `CTA_API_KEY` in
  the runtime `.env`.
- `nber_digest.py` — the weekly scraper; scores papers and writes `digest.json`.

**Runtime location.** macOS blocks background (launchd) apps from reading
`~/Documents`, so `install.sh` deploys a copy of the runtime to
`~/Library/Application Support/NBERDigest` (and builds the venv there). The git
repo in Documents is the *source* — **re-run `install.sh` after editing source
files** (e.g. `profile.md`) to redeploy.

Scoring is topic-dominant: area fit sets the tier, and author reputation only
breaks ties — Chicago/Booth economists get a larger bump than generally-famous
economists, and a famous off-topic paper never outranks an on-topic one.

## Setup (first machine)

```bash
cd nber_digest
./install.sh        # creates a venv, installs deps, asks for your API key, schedules the weekly job
```

You'll need an Anthropic API key (https://console.anthropic.com). It's stored in
`nber_digest/.env`, which is gitignored and never committed. Each run costs a few
cents.

Test it immediately:

```bash
.venv/bin/python nber_digest.py
```

## Moving to a new computer

The code and your interest profile live in this git repo, so:

```bash
git clone https://github.com/Poumzday/UChicago-Advisor-Search-.git
cd UChicago-Advisor-Search-/nber_digest
./install.sh        # re-creates the venv and the weekly schedule; re-enter your API key
```

The only machine-specific piece — the launchd schedule — is regenerated by
`install.sh`. Your API key is re-entered (it is never stored in git).

## Tuning

- **What counts as relevant:** edit `profile.md`.
- **Threshold:** change `SCORE_THRESHOLD` in `nber_digest.py` (default 80).
- **Schedule:** edit the `StartCalendarInterval` in `install.sh` and re-run it
  (`Weekday` 1 = Monday).
- **Model:** defaults to `claude-opus-4-8`; override with `NBER_MODEL`.

## Uninstall

```bash
for L in com.poum.nberdigest com.poum.nberdigest.menubar com.poum.nberdigest.weather; do
  launchctl bootout "gui/$(id -u)/$L" 2>/dev/null
  rm -f ~/Library/LaunchAgents/$L.plist
done
rm -rf ~/Library/Application\ Support/NBERDigest
```
